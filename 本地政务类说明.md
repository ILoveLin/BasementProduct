# 本地政务类功能总结与功能点描述

> 对赣政通、赣服通、E呼即办三大本地政务平台的功能总结，以及各功能点的描述说明。

---

## 一、赣政通 —— 政务协同办公平台

### 1.1 平台概述

赣政通是江西省统一政务协同办公平台，覆盖省-市-县-乡四级、联通超万个政务部门。它与赣服通形成"前店后厂"模式——赣服通前台受理、赣政通后台办理。

### 1.2 功能总结

- **任务分派**：创建任务→指派到人/部门→推送待办中心→跟踪进度→办结归档
- **公文流转**：收文全流程（签收→登记→审核→分办→拟办→批办→承办）线上完成
- **督查督办**：督办事项逐级下发→实时跟踪→超时自动催办→结果上报→核查销号
- **协同通讯**：政务通讯录、即时消息、工作群组、视频会议、通知公告
- **低代码搭建**：拖拽式搭建个性化微应用（审批、报销、考勤等）
- **安全体系**：政务云存储 + 国密算法加密 + 统一身份认证 + 全程操作留痕

### 1.3 功能点描述

| 功能点 | 描述 |
|--------|------|
| 待办中心 | 所有待办事项统一汇聚，按类型分类展示，支持红点提醒和消息推送。本平台产生的任务可推送至此 |
| 任务分派 | 指定执行人/部门、办理时限、任务要求，系统自动跟踪进度，超时逐级催办（执行人→主管→领导） |
| 公文流转 | 收文从2天以上压缩至最快1小时，文件办理从2-3周压缩至3-4天。全程留痕，各环节办理时长可追溯 |
| 督查督办 | 全国率先实现与国办"互联网+督查"系统贯通，省→市→县→乡逐级分派，支持核查销号闭环 |
| 统一身份认证 | 对接省统一身份认证平台，支持人脸识别、数字证书，可与本平台实现互信登录 |
| 低代码平台 | 各部门可拖拽式搭建内部微应用，如请假审批、公车申请、经费报销等 |

### 1.4 与本平台对接

| 对接项 | 说明 | 优先级 |
|--------|------|--------|
| 身份互信 | 通过省统一身份认证平台实现互信登录 | P0 |
| 待办同步 | 本平台任务自动推送至赣政通待办中心，签收/完成状态双向同步 | P0 |
| 消息通知 | 本平台预警、提醒通过赣政通消息通道推送 | P1 |
| 督查对接 | 超时未处置事件可同步至赣政通督查督办模块 | P1 |

## 二、赣服通 —— 群众办事移动端总入口

### 2.1 平台概述

赣服通是江西省集成全省各级政务服务资源的移动端总平台，覆盖查询、预约、申请、办理、投诉、评价等全流程。与赣政通形成"前店后厂"——赣服通是群众的"前店"，赣政通是政务人员的"后厂"。

### 2.2 功能总结

- **诉求上报**：群众扫码/搜索→选择问题类型→填写描述→上传照片/视频→提交
- **"码上反映"**：扫二维码门牌直达诉求提交页，地址自动获取无需手动输入
- **智能分办**：系统自动识别问题类型和地址，匹配责任网格/部门。简单问题智能应答，需处置问题生成工单派发至赣政通
- **进度追踪**：群众可实时查看办理进度，办结后收到通知
- **一事一评**：办结后群众打分评价，不满意自动进入"二次办理"
- **便民服务**：居住证申请、低保、补贴、物业缴费等政务服务掌上办理
- **多渠道整合**：12345热线、网上信访、"监督一点通"等诉求渠道统一汇聚

### 2.3 功能点描述

| 功能点 | 描述 |
|--------|------|
| 诉求提交 | 扫码门牌或搜索服务→选择类型→填写内容+上传证据→提交。地址由扫码自动获取，无需手动输入 |
| 智能分办 | 识别问题类型+地址→自动匹配责任部门/网格员→简单问题知识库自动回复→复杂问题生成工单推至赣政通 |
| 限时办结 | 一般问题5个工作日、复杂问题10个工作日办结，超时自动预警 |
| 满意度评价 | 办结后"一事一评"，不满意触发"二次办理"重新派单，"小赣事"帮办队伍（489人）回访跟踪 |
| 便民服务 | 居住证、低保、补贴等政务服务的在线申请→赣服通受理→赣政通审核→结果回传 |
| 支付通道 | 物业缴费、维修缴费等可通过赣服通支付通道完成 |

### 2.4 与本平台对接

| 对接项 | 说明 | 优先级 |
|--------|------|--------|
| 诉求同步 | 群众在赣服通提交的诉求自动同步至本平台事件中心 | P0 |
| 地址匹配 | 赣服通扫码门牌地址与本平台"一标三实"地址库自动匹配 | P0 |
| 结果回传 | 本平台处置结果（照片+时间戳）回传赣服通，供群众查询评价 | P0 |
| 一码共治 | 二维码门牌作为共同入口——群众扫码进赣服通，工作人员扫码进本平台 | P0 |
| 便民服务发布 | 本平台便民服务可作为赣服通服务项发布 | P1 |

## 三、E呼即办 —— 民生诉求快速响应平台

### 3.1 平台概述

E呼即办是上饶市创新建设的民生诉求快速响应平台，整合110、12345热线、社会治理网格三张网（47条热线统一归并至12345），18家高频部门常驻联合指挥中心，实现群众诉求"一号对外、统一受理、快速响应、闭环处置"。

### 3.2 功能总结

- **"12530"快速响应**：1分钟形成工单→2分钟派单→5分钟签收→30分钟到达现场→处置反馈→三级回访
- **三网融合**：110报警、12345政务热线、社会治理网格三张网统一归集、统一调度
- **智能分派**：依托标准地址编码+事件分类规则引擎，自动匹配责任单位
- **部门联动**：复杂事项启动"线上协同+线下联合+多元调解"机制
- **街乡吹哨**：乡镇遇到无权限问题→发起"吹哨"→指挥中心审核→相关部门"报到"→联合处置→吹哨单位评分
- **三级回访**：所有办结事项回访→不满意启动"二次化解"→仍不满意领导包案、提级调度
- **不满意复盘**：每日复盘9个环节（受理/分类/派单/响应/处置/反馈/办结/回访/考核），纪委联动追责，媒体曝光倒逼整改
- **未呼先办**：从高频共性问题中发现规律→修订制度流程→从源头减少同类诉求，变"被动响应"为"主动治理"

### 3.3 功能点描述

| 功能点 | 描述 |
|--------|------|
| 统一受理 | 47条热线统一归并至12345，群众一个电话即可触达所有政务服务。话务员1分钟内完成记录并生成标准化工单 |
| "12530"响应 | 1分钟成单→2分钟派单→5分钟签收→30分钟到场，平均办结时长从44小时压缩至12小时（缩短73%） |
| 智能派单 | 自动识别地址和问题类型，依托标准地址编码+事件分类规则引擎精准匹配责任单位和网格员 |
| 部门联动 | 跨部门复杂事项由联合指挥中心统筹，线上多部门会商+线下联合到场+多元调解力量参与（法律明白人、退休法官等） |
| 吹哨报到 | 乡镇发起吹哨→指挥中心审核→部门报到响应→联合处置→吹哨单位评分。已处理92件，延伸解决问题370余个，满意率96.5% |
| 三级回访 | 第一次全部回访→第二次不满意"二次化解"→第三次领导包案、提级调度，确保群众满意率98.6%以上 |
| 不满意复盘 | 每日复盘不满意工单的9个环节，纪委联动追责，广丰区特色为媒体"现场问政"曝光倒逼，高频问题由一把手亲自走流程 |
| 未呼先办 | 从海量工单中发现共性问题→修订制度/优化流程→从源头减少同类诉求，实现从"接诉即办"到"未呼先办"的升级 |

### 3.4 与本平台对接

| 对接项 | 说明 | 优先级 |
|--------|------|--------|
| 工单双向同步 | E呼即办工单自动同步至本平台事件中心；本平台网格员上报事件可推送至E呼即办 | P0 |
| 标准地址匹配 | E呼即办工单地址通过本平台"一标三实"底座自动解析、匹配网格和责任人 | P0 |
| 处置过程互通 | 网格员在本平台的处置记录（照片/轨迹/结果）实时回传E呼即办，群众可查 | P0 |
| 融合通讯联动 | E呼即办指挥调度时可调取本平台的无人机图传、AI眼镜直播流 | P1 |

## 四、三大平台关系

```
群众入口：赣服通（掌上办事） / 12345热线（电话诉求） / 二维码门牌（现场扫码）
                              ↓
指挥中枢：E呼即办（统一受理 → "12530"快速响应 → 三网融合调度）
                              ↓
后台办理：赣政通（任务推送/公文流转/督查督办） + 本平台（数据底座/协同处置）
                              ↓
                         结果反馈 → 群众评价
```

> **核心理念**：本平台不做新的孤岛，而是融入现有政务生态——赣服通是群众入口、E呼即办是指挥中枢、赣政通是办理后台，本平台是贯穿其中的**数据底座和协同处置工具**。
