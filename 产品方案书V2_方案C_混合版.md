# 《基层智治综合协同平台》产品方案书 V2.0 · 方案C（混合版）

> **版本说明：** 本版将**事件总线**融入基础平台（Platform）作为核心能力，将**安全与合规**拆分融入各层的描述中而非独立成章。1+N+1+N架构保持完全不变，不新增任何横向层或额外层。这是三种方案中**改动最小、最易通过评审**的版本。

#### 3.2.5 事件总线引擎（新增）

> 基层治理本质是事件驱动——发现→上报→分拨→处置→反馈→评价。

## 五、第二个"1"：AI赋能中心（AI）—— 辅助决策工作台

> **安全合规融入：** Agent合规控制作为AI赋能中心的独立子章节。

### 5.1 核心理念

> **AI只做建议、不做决定。业务决策权永远在一线业务员侧。**
> 每个Agent = 专题知识库 + 场景Skill，不是"通用AI"而是"懂这个领域的数字同事"。

### 5.2 五个专题Agent

| Agent | 专题知识库 | 场景Skill | 辅助决策方式 |
| --- | --- | --- | --- |
| **采集Agent** | 地址标准库、数据字典、校验规则库 | 语音转表单、OCR识别、对话引导 | AI填充→网格员确认→提交 |
| **巡查Agent** | 消防法规库、建筑规范库、隐患图谱 | 隐患视觉识别、AR引导、自动报告 | AI标记隐患→检查员确认→生成报告 |
| **调解Agent** | 法律条文库、案例库、策略模板库 | 案例检索、法规定位、策略建议 | AI推荐→调解员参考→自主判断 |
| **分拨Agent** | 部门矩阵、历史记录、负荷数据 | 事件识别、部门匹配、协同建议 | AI推荐→综治中心确认→下达 |
| **预判Agent** | 时序库、空间聚类、季节规律库 | 周期预测、热力分析、根因分析 | AI预测→业务部门研判→制定措施 |

### 5.3 Agent合规控制

| 控制机制 | 说明 |
| --- | --- |
| **输出过滤** | 文本经内容安全过滤——防范不当言论、错误引导、敏感信息泄露 |
| **建议可溯源** | 每条建议附带法规出处/案例编号/数据来源 |
| **闭眼兜底** | 准确率<85%→仅建议；<70%→自动暂停、人工优化后重新上线 |
| **决策在业务员** | 关键决策节点强制人工确认——不设"AI自动决策"流程 |
| **法规定期对齐** | 法规变化→自动解析→知识库更新→Agent重新校准（季度/次） |

## 六、"N"个场景：已落地三区县 + 上饶专属场景 + "一件事"叙事

> **安全合规融入：** 场景合规作为场景描述的"前提条件"呈现。

**合规前提（所有场景共用）：** AI行为识别不采集个体身份；校园场景保护未成年人数据；养老场景保护特殊群体数据脱敏；各上饶专属场景的合规要求由主管部门在场景上线前完成评审。

### 6.5 "一件事"叙事

#### "消防检查一件事"
过去：40分钟/商铺，漏项率15% → 现在：AI眼镜扫码→AR引导→AI识别隐患→自动报告，15分钟/商铺，漏项率<2%

#### "矛盾纠纷调解一件事"
过去：短则3天长则2周 → 现在：扫码获取背景→AI推送案例+法规→远程专家支援→在线签署调解文书，10-30分钟化解

#### "独居老人关爱一件事"
过去：出事几小时至几天才发现 → 现在：北斗轨迹异常→自动生成关爱任务→网格员上门→从预警到送医42分钟

#### "违建治理一件事"
过去：人工巡查→层层审批→处置周期长 → 现在：无人机自动发现→AI识别→工单自动生成→地面核查→复飞验证，周期缩短66%

## 十、总结与展望

"1+N+1+N"架构以基层治理平台为统一底座（五大共性能力含事件总线引擎），以生态平台（三级分类）为能力插件，以AI赋能中心（5个专题Agent=知识库+Skill，辅助决策）为智慧大脑，灵活支撑N个治理场景（"一件事"叙事+上饶专属场景）。

**安全与合规不另起章节——融入每一层的设计之中。** 信创适配和等保密评作为全平台基线要求。
