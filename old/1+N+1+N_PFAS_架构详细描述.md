# "1+N+1+N"（PFAS）架构详细描述

> 适用文档：《基层智治综合协同平台》产品方案书 · 第二章 · 总体架构

---

## 一、架构命名与全称

**"1+N+1+N"** 是一个分层解耦、能力复用的四级架构体系，英文缩写为 **PFAS**：

| 层级 | 数量 | 字母 | 英文全称 | 中文名称 | 角色隐喻 |
|------|------|------|----------|----------|----------|
| 第一层 | **1** | **P** | **P**latform | 基础平台 | 地基 |
| 第二层 | **N** | **F** | **F**ederation | 生态平台 | 插件 |
| 第三层 | **1** | **A** | **A**I | AI赋能中心 | 大脑 |
| 第四层 | **N** | **S** | **S**cenarios | 治理场景 | 屋顶 |

**一句话概括：** 用**一个厚底座**沉淀共性能力、用**N个开放插件**联接内外生态、用**一个智慧大脑**贯通智能、用**N个敏捷场景**靶向服务，构建"能力可复用、生态可扩展、智能可注入、场景可组装"的治理数字化大厦。

---

## 二、四层架构详述

### 第一层：1个基础平台（Platform）—— 共性能力中台

**定位：** 整个体系的统一底座，由公司自主研发，承担"能力沉淀、数据汇聚、标准统一、服务开放"四项核心职责。所有上层能力必须通过底座才能访问数据、调用服务、触达用户。

**核心设计理念：** 不让每个场景从零开始——把六大共性能力一次性做扎实。

#### 六大功能模块

##### 1. 统一门户与身份认证

- **五级组织架构**：省—市—县（区）—乡镇（街道）—村（社区）五级组织一体化实名管理
- **统一身份认证**：对接江西省统一身份认证平台，支持人脸识别、数字证书等多因素认证
- **互信登录**：与赣政通实现互信登录，用户一个账号通两平台
- **权限精细化管控**：基于"角色+辖区+数据分级"的权限模型，网格员只能看本网格数据
- **多端适配**：支持PC端、移动端（APP/小程序）、AI眼镜端统一登录

##### 2. 数据中台与治理数据湖

- **一标三实数据模型**：以标准地址（一标）为锚点，串联实有人口、实有房屋、实有单位（三实），形成"人—房—事—物—组织"五大要素标准化数据模型
- **多源数据汇聚**：汇聚公安、综治、住建、城管、民政等多部门数据，形成辖区全域"块数据"
- **数据治理与清洗**：地址标准化（GB/T 35639-2017）、身份证号校验、重复数据合并、失效数据标记
- **数据鲜活度管理**：通过AI眼镜巡查、网格员采集、群众自主申报等多渠道持续更新数据
- **数据资产沉淀**：在治理运行中自然累积部门处置"运行数据"和地址下的人口、房屋、事件"块数据"，为远期数据产品化奠定基础

##### 3. 时空地理信息GIS底座

- **统一地图服务**：二维/三维地图渲染、卫星影像切换、POI搜索、路径导航
- **标准地址编码**：基于国标建立全市统一地址编码体系，与二维码门牌一一对应
- **一张图作战**：多图层叠加——人口热力、事件分布、风险等级四色图、无人机飞行轨迹、AI眼镜实时位置
- **空间分析**：缓冲区分析、空间聚合、轨迹回放
- **无人机航线管理**：在GIS地图上规划、预览、管理自动巡检航线
- **三维数字底座**：重点区域厘米级精度三维建模，支撑无人机航拍3D建模和AR标注

##### 4. 智能能力开放平台

- **AI服务统一接入**：大模型、OCR、语音识别、图像识别等AI能力统一接入和管理
- **API开放网关**：将AI能力封装为标准化API，供生态平台和治理场景按需调用
- **调用监控**：API调用量、响应时间、成功率实时监控，异常告警
- **能力编排**：支持将多个AI能力串联为复合服务（如"语音→转写→语义分析→自动填单"）

##### 5. 周边（地址关联信息智能推送）

- **以地址为锚点的智能推送**：扫码门牌后，基于标准地址自动关联并推送该地址周边的历史隐患、重点关注人员、独居老人、出租房状态等信息
- **角色差异化呈现**：民警扫酒店门牌→自动弹出历史消防隐患和警情记录；网格员走访→自动提醒该户是否为独居老人/重点关注对象；城管扫码→推送该商铺占道经营历史
- **数据跨部门关联**：基于"一标三实"数据模型，将公安、消防、城管、民政等部门的记录以地址为锚点关联，扫一个码即可看到该地址的"全息画像"
- **落地效果**：巡查针对性提升60%，现场信息查询时间从平均5分钟降至"扫码即得"

##### 6. 格友圈/协同圈（跨部门协同与居民互动）

- **格友圈（工作人员端）**：网格员发现问题后，一键将工单共享至"格友圈"——城管、市政、市监等相关部门在同一事件页面内协同处置，打破"一个人解决不了"的困境
- **邻里圈（居民端）**：居民扫码进入本小区"邻里圈"——发布邻里互助、二手交易、物业报修，形成"居民自治+平台支撑"的社区互动生态
- **跨部门协同闭环**：格友圈内事件处置过程全员可见、全程留痕，相关部门响应时效和处置结果公开透明
- **落地效果**：跨部门协同周期从3.5天缩短至1.2天（缩短66%），居民"零跑腿"事项达12项

### 第二层：N个生态平台（Federation）—— 能力与联接延伸

**定位：** 本平台与外部世界联接的桥梁。遵循"开放对接、为我所用、不重新发明"的原则，既引入市场上优秀智能硬件的能力，又深度对接本地政务系统，让本平台不成为**新的孤岛**，而是现有政务生态的"**赋能枢纽**"和"**黏合剂**"。

**核心设计理念：** 联接而非替代——市场上已有的优秀硬件和政务系统，不做重复建设。

#### 2.1 智能硬件类（市场优秀产品能力引入）

##### AI眼镜

| 维度 | 说明 |
|------|------|
| 核心能力 | 解放双手的巡查记录工具——语音上报、远程标注、AI识别可疑人员/车辆、身份证OCR |
| 杀手场景 | 安全巡检（AR引导检查清单+AI隐患识别，效率提升62.5%，漏项率从15%降至<2%） |
| 协同价值 | 与无人机组成"空地一体"立体巡防：天上看、地上查，前后方信息实时贯通 |
| 对接方式 | 通过标准化API接入眼镜SDK，品牌无关化接入层，切换供应商仅需适配硬件层 |

##### 二维码门牌

| 维度 | 说明 |
|------|------|
| 核心能力 | 为每栋房屋、每个单位赋予唯一的"数字身份证"，是物理世界与数字世界的连接入口 |
| 三重入口 | 群众扫码→赣服通（便民服务）；工作人员扫码→本平台（工作入口）；管理者扫码→数据看板（辖区概况） |
| 协同价值 | "一码共治"——多部门扫同一个码，各取所需：民警查人口、物业发公告、城管看历史违建 |
| 对接方式 | 标准地址编码（GB/T 35639-2017）与门牌二维码中的地址ID一一对应 |

##### 电子地图

| 维度 | 说明 |
|------|------|
| 核心能力 | GIS时空底座，支撑二维/三维地图渲染、图层叠加、空间分析、实时位置服务 |
| 协同价值 | 为无人机航线规划、AI眼镜位置展示、事件热力图、风险分色图提供统一地理底板 |
| 对接方式 | 政务内网侧用天地图，互联网便民侧用高德/百度地图SDK；空间数据库用PostgreSQL+PostGIS |

##### 无人机（自动巡飞与AI识别）

| 维度 | 说明 |
|------|------|
| 核心能力 | 自动机库+航线规划+自动巡飞+AI智能识别+闭环工单联动 |
| 识别场景 | 分三期建设，覆盖违建、垃圾、火点、占道、河道异常、工地裸土等16+类 |
| 协同价值 | 与AI眼镜组成"天上看+地上查"闭环；发现的工单自动流入事件中心、分拨至网格员 |
| 对接方式 | 通过无人机管理调度中台统一接入，不做品牌绑定 |

##### 融合通讯终端

| 维度 | 说明 |
|------|------|
| 核心能力 | 视频会议终端、公网对讲机、执法记录仪等通讯设备的统一接入 |
| 协同价值 | 通过SIP/H.323标准协议接入基础平台，实现跨设备、跨网络的统一指挥调度 |
| 对接方式 | 基于基础平台的通讯基础设施，支持各归口部门终端统一接入 |

#### 2.2 本地政务类（深度对接本地工作平台）

##### 赣政通（任务推送）

| 维度 | 说明 |
|------|------|
| 平台定位 | 江西省统一政务协同办公平台，"前店后厂"模式中的"后厂" |
| 对接内容 | 统一身份互信登录；本平台任务自动推送至赣政通待办中心；签收/完成状态双向同步 |
| 协同价值 | 基层人员在赣政通一个入口处理所有待办，无需登录多个系统 |

##### E呼即办（民生诉求联动）

| 维度 | 说明 |
|------|------|
| 平台定位 | 上饶市创新民生诉求快速响应平台，"12530"机制整合三张网（110/12345/社会治理网格） |
| 对接内容 | 工单双向同步；一标三实底座自动地址解析+匹配网格；处置过程互通；融合通讯联动 |
| 协同价值 | E呼即办是"前台指挥"，本平台是"后台数据+处置协同"，双平台联动互补 |

### 第三层：1个AI赋能中心（AI）—— 平台级智慧大脑

**定位：** AI赋能中心不是孤立的AI功能模块，而是**贯穿各层的平台级智慧引擎**。它将大模型、OCR、语音识别等AI能力注入基础平台、生态平台和治理场景的每个环节，让知识"自动工作"，是近期基层减负最直接的抓手。

**核心设计理念：** ①AI始终定位为"辅助决策"，业务决策权永远在一线人员手中；②AI能力以**专题Agent**形态具象化（"矛盾纠纷调解Agent""违建巡查Agent""流程审核Agent"等），每个Agent背后是真实的专题知识库+场景skill，而非泛泛的"通用智能"。

#### 四大核心功能

##### 1. 智能案例编写与推荐

| 维度 | 说明 |
|------|------|
| 能力描述 | 网格员语音简述事件经过 → AI自动匹配历史相似案例 → 生成简报初稿 → 人工确认发布 |
| 解决痛点 | 告别手写报告，改变"不会写、不愿写、写得慢"的现状 |
| 减负效果 | 每人每天节省约40分钟报告撰写时间 |
| 技术支撑 | 语音转写（ASR）+ 语义理解（NLU）+ 案例相似度匹配（向量检索）+ 文本生成（LLM） |

##### 2. 知识库智能问答

| 维度 | 说明 |
|------|------|
| 能力描述 | 工作人员语音提问 → AI从政策法规、办事流程、历史案例知识库中检索答案 → 秒级返回 |
| 解决痛点 | 基层人员"不知道政策依据""不知道该找哪个部门""新人上手慢" |
| 减负效果 | 新人培训周期从1个月缩短至1周；现场政策查询正确率从60%提升至90% |
| 技术支撑 | RAG（检索增强生成）+ 语义向量索引 + 多轮对话 |

##### 3. 制度库自动校验

| 维度 | 说明 |
|------|------|
| 能力描述 | 事件分拨、结案时AI自动比对制度库中的办理时限、材料格式、审批层级要求 → 不合规自动提醒 |
| 解决痛点 | 基层"超期不知情""材料不合规反复退件""审批越级或漏级" |
| 减负效果 | 同类事件处置一致性提升至85%；事件超时率大幅下降 |
| 技术支撑 | 规则引擎 + LLM语义理解 + 流程状态机自动化比对 |

##### 4. 治理数据洞察与预判（远期）

| 维度 | 说明 |
|------|------|
| 能力描述 | 深度挖掘治理运行数据 → 自动发现规律 → 生成风险预测 → 提出干预建议 |
| 远期价值 | 区域风险预测、季节性事件预警、资源投放优化建议 → 最终形成可对外销售的数据产品 |
| 技术支撑 | 时序分析 + 空间聚类 + 因果推断 + LLM辅助解读 |

### 第四层：N个治理场景（Scenarios）—— 靶向解决一线痛点

**定位：** 治理场景层是**面向角色、面向业务的复合应用层**。它不是独立的子系统，而是基于基础平台能力+AI赋能+生态平台能力，按"用户角色+核心业务"组合而成的综合解决方案。新增场景通过"五大要素判定法"快速匹配和开发。

**核心设计理念：** 既有统一底座，又不束缚灵活创新——匹配到已有模块→直接复用；有共性但无模块→抽象为通用能力；无共性→独立开发。

#### 四大核心场景

##### 公安治理

- **目标用户：** 民警、辅警、派出所人员
- **核心功能：** AI眼镜+无人机立体巡防、异常自动比对；二维码门牌出租屋扫码登记；重点人员管控与轨迹追踪；治安态势一张图实时监测
- **关键协同：** AI眼镜（巡查）、无人机（高空监控）、二维码门牌（入户登记）、赣政通（任务派发）

##### 住建物业管理

- **目标用户：** 住建工作人员、物业人员、业主
- **核心功能：** 打通维修基金平台，居民扫码报修；AI智能派单、资金透明流转；无人机巡查建筑安全/工地裸土
- **关键协同：** 房屋维修基金平台、二维码门牌（扫码报修）、无人机（工地巡查）

##### 综治网格员

- **目标用户：** 网格员、综治工作人员、便民服务员
- **核心功能：** 语音上报、AI填单、自动生成工单；任务自动分拨与绩效考核；二维码门牌"一次采集多方复用"
- **关键协同：** AI赋能中心（语音转写/案例生成）、赣政通（任务推送）、二维码门牌（采集入口）

##### 街道社区

- **目标用户：** 社区干部、社工、群众
- **核心功能：** 一块屏统览辖区态势（分色图、事件热力）；对接E呼即办快速响应民生诉求；AI自动生成治理月报
- **关键协同：** E呼即办（诉求联动）、AI赋能中心（月报生成）、数据中台（态势数据）

#### 场景扩展机制：五大要素判定法

| 要素 | 判定内容 | 示例 |
|------|---------|------|
| 用户要素 | 谁在用？（角色+权限） | 河长 → 河道巡查场景 |
| 地址要素 | 在哪用？（标准地址来源） | 扫码河段二维码→获取河段编码 |
| 业务意图 | 要干什么？（操作类型） | 巡查→记录水位、拍照、上报污染 |
| 前置状态 | 能用吗？（数据/流程前提） | 当前时段在巡查排班计划内 |
| 设备要素 | 用什么设备？ | 手机必须（GPS）+ AI眼镜推荐 + 无人机备选 |

## 三、四层之间的纵向关系与数据流

```
治理场景（N）    ←  面向角色呈现，调用下层能力
    ↕ 调用/反馈
AI赋能中心（1）   ←  横贯各层：向上赋能场景、向下注入底座、左右联接生态
    ↕ 调用/反馈
生态平台（N）    ←  横向扩展：硬件能力接入 + 政务系统联接
    ↕ 调用/反馈
基础平台（1）    ←  纵向支撑：数据、GIS、通讯、流程、AI六个统一能力底座
```

### 关键数据流

| 流向 | 说明 | 示例 |
|------|------|------|
| 群众→平台 | 通过赣服通、二维码门牌、E呼即办三种入口提交诉求 | 扫码报修→生成工单→流入事件中心 |
| 平台→基层 | 赣政通任务推送、本平台APP消息通知 | 指挥中心派单→网格员收到"走访独居老人"任务 |
| 基层→平台 | 巡查采集数据、事件处置反馈 | AI眼镜拍照+语音上报→写入一标三实数据湖 |
| 平台←→生态 | 与外部平台双向数据同步 | E呼即办工单同步→本平台地址匹配→处置结果回传E呼即办 |
| AI贯穿 | 在各环节注入智能服务 | 语音→转写→语义分析→自动填单→案例推荐→月报生成 |

## 四、四个核心设计理念

### ① 厚底座 —— 不让每个场景从零开始

第一个"1"把六大共性能力（门户认证、数据中台、GIS底座、AI开放平台、周边、格友圈）一次性做扎实。过去公安、综治、住建各建系统，同一栋楼在不同系统里地址写法都不一样。现在以国标地址编码为锚点，将"人、房、事、物、组织"五要素统一建模，实现**一次采集、多方复用**，从源头终结数据孤岛。

### ② 活生态 —— 联接而非替代

第一个"N"承认两个事实：市场上已有优秀的智能硬件，政务侧已有成熟的业务系统。本平台不做重复建设，通过标准化接口实现"即插即用"——横向联接8类智能硬件（AI眼镜、无人机、二维码门牌、电子地图、融合通讯终端、IoT传感器、AI摄像头、北斗终端），纵向对接2大政务系统（赣政通、E呼即办）。让本平台成为现有政务生态的赋能枢纽，而非数据黑箱。

### ③ 强智能 —— 贯穿四层的智慧引擎

第二个"1"不是孤立的功能模块，而是以**专题Agent**形态注入每一层的平台级智慧。网格员语音描述→AI自动生成简报；事件分拨→AI自动校验制度库；长期运行→AI主动预判风险。每个Agent背后是真实的专题知识库+场景skill。AI始终定位为**辅助决策**，业务决策权永远在一线人员手中。

### ④ 多场景 —— 统一底座之上的灵活应变

第二个"N"面向一线真实需求，通过**五大要素判定法**让每个新需求快速找到可复用模块。匹配到已有模块→直接复用；有共性但无模块→抽象为通用能力；无共性→独立开发。做到"既有统一底座，又不束缚灵活创新"。

## 五、核心理念

> **赋能而非替代** —— 本平台不替代任何现有部门业务系统，只做"黏合剂"和"减压阀"。
>
> 让公安的系统继续管公安的数据，综治的流程继续走综治的审批。
>
> 本平台做的是：用**标准地址**把各系统的数据串起来，用**AI**把重复劳动替掉，用**生态**把外部能力接进来。

## 六、服务目标映射

| 目标 | 对应架构层 | 实现路径 |
|------|-----------|---------|
| **基层减负** | AI赋能中心 + 周边（智能推送） | AI自动生成取代手写报告；一次填报多方复用减少重复录入 |
| **协同治理** | 基础平台（格友圈/数据中台）+ 生态平台（赣政通/E呼即办） | 统一地址模型串联各部门；"吹哨报到"跨部门联动 |
| **数据要素** | 基础平台（数据中台/一标三实） | 在治理运行中自然沉淀标准化"块数据"，完成原始积累 |
| **数字经济** | AI赋能中心（数据洞察）+ 基础平台（开放平台） | 治理数据产品化；API开放吸引生态，催生产业集群 |
